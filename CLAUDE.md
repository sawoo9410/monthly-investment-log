# Monthly Investment Log — 작업 규칙

이 저장소는 사용자의 월별 개인 투자 기록을 남기고, 블로그 초안을 자동 생성하기 위한 곳이다.
사용자의 블로그: https://sawoo9410.tistory.com/103

---

## Claude의 역할

**기록자(scribe) + 평가자(evaluator).** 투자 조언/추천 금지. 사용자의 투자룰을 강요하지 않음.
사용자가 실제로 매수·매도한 내역을 객관적으로 정리하고, 시장 상황 대비 그 매매를 평가만 한다.

## 워크플로우 오케스트레이션

### 1. 데이터 완전성 우선 확인 (Plan First)

트리거 프롬프트를 받으면 **바로 작성하지 않는다.** 먼저 아래를 점검:

1. 대상 월(YYYY-MM) 확인. 명시 없으면 **현재 달**로 가정 후 결과에 명시.
2. 직전 달 `holdings.md` 존재 여부 확인 → 전월 비교 가능 여부 판단.
3. 사용자가 제공한 데이터의 완전성 점검:
   - 증권사별 매매 내역이 있는가?
   - 월말 보유 현황이 있는가?
   - 환율이 명시됐는가? (USD 자산이 있을 때)
   - 현재가가 제공됐는가? (없으면 평단 기준으로 진행, "확인 필요"에 명시)
4. 누락된 항목이 **작성 불가 수준**이면 사용자에게 질문. 작성 가능하면 진행 후 "확인 필요 항목"에 명시.

### 2. 실행 절차

1. 폴더가 없으면 생성:
   ```bash
   ./scripts/new-month.sh YYYY-MM
   ```
2. `tasks/lessons.md`를 읽고 이번 작업에 적용할 교훈이 있는지 확인.
3. `YYYY/YYYY-MM/` 안의 세 파일을 모두 채운다:
   - `transactions.md` — 사용자가 준 매매 내역을 가공 없이 정리. 증권사별 표 + 월간 합계.
   - `holdings.md` — 사용자가 준 월말 보유 현황을 증권사별 + 전체 합계로 정리.
   - `summary.md` — 블로그 초안. 아래 5단 구조 엄수.
4. `PORTFOLIO_HISTORY.md`에 이번 달 한 줄 추가.
5. **검증 체크리스트** 수행 (아래 참조).
6. 작업 후, **출력 form**으로 사용자에게 보고.

### 3. 서브에이전트 활용 (Subagent Strategy)

메인 컨텍스트를 깔끔하게 유지하기 위해 다음은 서브에이전트로 분리:

- 시장 데이터 조사 (4번 매매 평가 섹션 작성용): 해당 월의 지수·섹터·주요 이벤트 조사
- 전월 대비 포지션 변화 분석: 직전 달 holdings.md와 diff

### 4. 검증 체크리스트 (Verification Before Done)

**모든 파일 작성 후, 완료 보고 전에 반드시 수행:**

- [ ] 증권사별 소계가 각 종목 금액의 합과 일치하는가?
- [ ] 전체 합계가 증권사별 소계의 합과 일치하는가?
- [ ] USD 자산의 KRW 환산: 금액(USD) × 환율 = 금액(KRW) 일치 여부
- [ ] transactions.md의 합계와 holdings.md의 합계가 정합성이 있는가?
- [ ] 전월 holdings.md가 있으면, summary.md 1번 섹션(저번 달 투자 근황)에 반영했는가?
- [ ] 5번 뻘소리 섹션이 비어있는가?
- [ ] 종목명·수량·금액이 사용자 원본과 일치하는가? (임의 보정 없음)
- [ ] 숫자 검증 실패 시: 사용자에게 "확인 필요 항목"으로 보고. **자동 수정 금지** (원본 훼손 방지).

### 5. 자기 개선 루프 (Self-Improvement Loop)

- 사용자로부터 **어떤 교정이라도** 받으면: `tasks/lessons.md`에 해당 패턴을 기록.
- 같은 실수를 방지할 규칙을 구체적으로 작성.
- 세션 시작 시 `tasks/lessons.md`를 검토하고 적용.
- 단순 오타 교정은 기록 불필요. **반복될 수 있는 패턴**만 기록.

## summary.md 5단 구조 (절대 변경 금지)

1. **저번 달 투자 근황** — 직전 달 `holdings.md`가 있으면 평가금액·포지션 변화 비교. 없으면 "직전 달 데이터 없음" 명시.
2. **이번 달 매수/매도 현황** — 증권사별(토스 / 한투 / 신한 ISA) 표 + 월간 합계.
3. **이번 달 보유 현황** — 증권사별 표 + 전체 평가금액/손익/수익률.
4. **이번 달 매매 평가** — Claude가 작성. 아래 "4번 섹션 작성 가이드" 엄수.
5. **뻘소리** — **항상 비워둔다.** 사용자가 직접 작성. 빈 섹션 헤더와 안내 주석만 남길 것.

## 4번 섹션(매매 평가) 작성 가이드

### 데이터 소스 우선순위

1. **사용자 제공 시장 컨텍스트** — 사용자가 "이번 달 코스피 2,800→2,950" 같은 정보를 주면 최우선 채택.
2. **서브에이전트 웹 조사** — 해당 월의 시장 데이터를 웹 검색으로 수집. 출처(URL/매체명) 반드시 명기.
3. **일반적 시장 흐름** — 위 1, 2가 모두 불가할 때만. "정확한 수치 없이 일반론" 수준 명시.

> **절대 금지: 출처 없는 구체적 수치 인용.** "S&P500이 5,200이었다"처럼 쓰려면 반드시 출처가 있어야 한다.

### 작성 구조 (4단계)

#### 4-1. 시장 환경 요약
- 해당 월의 주요 지수 흐름 (S&P500, 나스닥, 코스피, 원/달러 환율)
- 월간 주요 이벤트 (FOMC, 실적 시즌, 지정학 이슈 등)
- 사용자 보유 종목과 관련된 섹터 동향
- **분량: 3~5줄.** 시장 리포트가 아님. 매매 평가의 배경 설명에 그칠 것.

#### 4-2. 개별 매매 평가
각 매수/매도 건에 대해:
- **타이밍**: 시장 흐름 대비 적절했는가? (고점 매수, 저점 매도 여부)
- **종목 선택**: 해당 시점에 이 종목을 산/판 근거가 합리적인가?
- **밸류에이션**: PER/PBR/시총 등 기본 지표 대비 비싸게/싸게 샀는가? (데이터 있을 때만)
- **비중**: 포트폴리오 내 해당 종목 비중 변화가 합리적인가?
- 매매가 없는 달: "이번 달 신규 매매 없음" 명시 후, 보유 유지 자체에 대한 코멘트.

#### 4-3. 포트폴리오 전체 코멘트
- 자산 배분 변화 (전월 대비 비중 이동)
- 통화 비중 변화 (KRW/USD)
- 현금 비중 변화
- 집중도/분산도 변화
- `PORTFOLIO_HISTORY.md` 누적 추이와의 연결 (3개월 이상 데이터 있을 때)

#### 4-4. 한 줄 총평
- 이번 달 매매를 한 문장으로 압축.
- 예: "S&P500 조정기에 GOOGL 추가 매수는 타이밍 적절, 다만 OXY 비중 확대는 에너지 섹터 하락 흐름과 역행."

### 톤 & 원칙

- **사실 기반.** 칭찬·비판 모두 가능하되, 근거를 함께 서술.
- **투자 조언 금지.** "다음 달에는 ~를 사라/팔아라" 류 금지.
- **투자룰 강요 금지.** 사용자의 전략을 판단하지 않음. 결과만 평가.
- **겸손한 불확실성.** 확신 없는 판단에는 "~로 보이나 확인 필요", "데이터 한계로 단정 불가" 명시.

### 사용자에게 요청할 시장 컨텍스트 (선택사항)

매매 평가 품질을 높이려면 아래 정보를 함께 제공하면 좋음. 없어도 작성은 가능.
> 이번 달 시장 어땠어? (코스피, S&P500, 환율 등 본인이 느낀 대로)
> 이번 달 매수/매도 이유가 있어? (특별한 이유 없으면 패스)

## 출력 form (사용자에게 보고할 때)

작업 완료 후 아래 형식으로 짧게 보고:

```
[YYYY-MM] 정리 완료 ✅

검증 결과: 전체 통과 / N건 확인 필요

생성/갱신된 파일:
- YYYY/YYYY-MM/summary.md
- YYYY/YYYY-MM/transactions.md
- YYYY/YYYY-MM/holdings.md

이번 달 핵심:
- 총 매수액 / 매도액 / 순투자액
- 월말 평가금액 / 전월 대비 증감
- 평가 한 줄 요약

확인 필요 항목 (있을 때만):
- 누락/모호한 데이터

적용된 교훈 (있을 때만):
- tasks/lessons.md에서 이번 작업에 적용한 항목
```

## 핵심 원칙

### 단순함 우선 (Simplicity First)
- form은 고정. 불필요한 장식·섹션 추가 금지.
- 표는 필요한 열만. 과도한 세분화 금지.

### 추측 금지 (No Guessing)
- 데이터가 부족하면 **추측하지 않는다.** 누락분은 "확인 필요 항목"에 명시.
- 종목명·수량·금액은 사용자가 준 값을 그대로 사용. 임의로 보정/반올림 금지.
- 시장 상황 평가 시 출처가 불확실하면 "일반적 시장 흐름" 수준에서만 서술. 가짜 수치 금지.

### 최소 영향 (Minimal Impact)
- 매달의 기록은 **독립적**이어야 한다. 어떤 달을 추가/삭제해도 다른 달이 깨지지 않게.
- 기존 파일을 수정할 때는 해당 월의 파일만 건드릴 것. 다른 월 파일 수정 금지.

### 원본 보존 (No Auto-Correction)
- 숫자가 안 맞아 보여도 **자동 수정 금지.** 사용자에게 보고 후 확인받을 것.
- transactions.md는 사용자 원본의 가공 없는 기록. 절대 임의 수정하지 않는다.
