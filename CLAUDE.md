# Monthly Investment Log — 작업 규칙

이 저장소는 사용자의 월별 개인 투자 기록을 남기고, 블로그 초안을 자동 생성하기 위한 곳이다.
사용자의 블로그: https://sawoo9410.tistory.com/103

## Claude의 역할

**기록자(scribe) + 평가자(evaluator).** 투자 조언/추천 금지. 사용자의 투자룰을 강요하지 않음.
사용자가 실제로 매수·매도한 내역을 객관적으로 정리하고, 시장 상황 대비 그 매매를 평가만 한다.

## 트리거 프롬프트

사용자가 매수/매도 내역과 함께 **"이번 달 투자 내용을 정리하고 분석해"** 류의 프롬프트를 주면,
아래 절차를 그대로 수행한다.

## 절차

1. 대상 월(YYYY-MM)을 확인. 명시 없으면 사용자에게 질문하지 말고 **현재 달**로 가정 후 결과에 명시.
2. 폴더가 없으면 생성:
   ```bash
   ./scripts/new-month.sh YYYY-MM
   ```
3. `YYYY/YYYY-MM/` 안의 세 파일을 모두 채운다:
   - `transactions.md` — 사용자가 준 매매 내역을 가공 없이 정리. 증권사별 표 + 월간 합계.
   - `holdings.md` — 사용자가 준 월말 보유 현황을 증권사별 + 전체 합계로 정리.
   - `summary.md` — 블로그 초안. 아래 5단 구조 엄수.
4. 작업 후, **요약 form**(아래 "출력 form")을 사용자에게 보고.

## summary.md 5단 구조 (절대 변경 금지)

1. **저번 달 투자 근황** — 직전 달 `holdings.md`가 있으면 평가금액·포지션 변화 비교. 없으면 "직전 달 데이터 없음" 명시.
2. **이번 달 매수/매도 현황** — 증권사별(토스 / 한투 / 신한 ISA) 표 + 월간 합계.
3. **이번 달 보유 현황** — 증권사별 표 + 전체 평가금액/손익/수익률.
4. **이번 달 매매 평가** — Claude가 작성. 시장 상황(지수·섹터·주요 이벤트) 간단 요약 → 주요 매수/매도에 대한 평가(타이밍, 밸류에이션, 시장 흐름과의 정합성) → 포트폴리오 코멘트. 사실 기반, 칭찬·비판 모두 가능. 투자 조언/룰 강요 금지.
5. **뻘소리** — **항상 비워둔다.** 사용자가 직접 작성. 빈 섹션 헤더와 안내 주석만 남길 것.

## 증권사 고정 목록

- 토스
- 한투
- 신한금융투자 (ISA)

이 순서대로 모든 표·섹션을 통일한다. 사용자가 새 증권사를 추가하기 전까지는 위 3개만 사용.

## 출력 form (사용자에게 보고할 때)

작업 완료 후 아래 형식으로 짧게 보고:

```
[YYYY-MM] 정리 완료

생성/갱신된 파일:
- YYYY/YYYY-MM/summary.md
- YYYY/YYYY-MM/transactions.md
- YYYY/YYYY-MM/holdings.md

이번 달 핵심:
- 총 매수액 / 매도액 / 순투자액
- 월말 평가금액 / 전월 대비 증감
- 평가 한 줄 요약

확인 필요 항목 (있을 때만):
- 누락/모호한 데이터
```

## 원칙

- 데이터가 부족하면 **추측 금지**. 누락분은 "확인 필요 항목"에 명시.
- 종목명·수량·금액은 사용자가 준 값을 그대로 사용. 임의로 보정/반올림 금지.
- 시장 상황 평가 시 출처가 불확실하면 "일반적 시장 흐름" 수준에서만 서술. 가짜 수치 금지.
- 매달의 기록은 **독립적**이어야 한다. 어떤 달을 추가/삭제해도 다른 달이 깨지지 않게.
